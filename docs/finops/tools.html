Implementation Guides

AWS Cost Monitoring Setup

Step-by-Step AWS Cost Explorer Setup

Enable Cost Explorer: AWS Console → Billing → Cost Explorer
Configure Cost Allocation Tags: Billing → Cost Allocation Tags → Activate required tags
Create Budgets: Billing → Budgets → Create budget (cost, usage, or reservation)
Set up Cost & Usage Reports: Billing → Cost & Usage Reports → Create report
Configure Alerts: Budgets → Create alert for budget thresholds
Enable Cost Anomaly Detection: Billing → Cost Explorer → Cost Anomaly Detection

Kubecost Installation

Install Kubecost on Kubernetes

Add Helm repository: helm repo add kubecost https://kubecost.github.io/cost-analyzer/
Create namespace: kubectl create namespace kubecost
Install Kubecost: helm install kubecost kubecost/cost-analyzer -n kubecost
Get service URL: kubectl get svc -n kubecost
Access dashboard: Open browser to Kubecost service URL
Configure cloud integration for accurate pricing data

Integration Patterns

Slack Integration for Cost Alerts

# AWS Lambda function for Slack notifications
import json
import urllib3
import os

http = urllib3.PoolManager()
SLACK_WEBHOOK = os.environ['SLACK_WEBHOOK_URL']

def lambda_handler(event, context):
    message = {
        "text": f"🚨 Cost Alert: {event['message']}",
        "attachments": [
            {
                "color": "danger",
                "fields": [
                    {"title": "Service", "value": event.get('service', 'N/A')},
                    {"title": "Amount", "value": event.get('amount', 'N/A')},
                    {"title": "Threshold", "value": event.get('threshold', 'N/A')}
                ]
            }
        ]
    }

    http.request('POST', SLACK_WEBHOOK,
                body=json.dumps(message),
                headers={'Content-Type': 'application/json'})

    return {'statusCode': 200}

Tool Selection Guide

Choosing the Right FinOps Tools

Startups & Small Teams
  • Cloud provider native tools (Cost Explorer, etc.)
  • Open-source options (Kubecost, Prometheus)
  • Simple budgeting and alerting
  • Focus on cost visibility first
Growing Companies
  • Third-party SaaS platforms
  • Advanced automation and optimization
  • Multi-cloud cost management
  • Custom dashboards and reporting
Enterprise Organizations
  • Enterprise FinOps platforms
  • Custom integrations and APIs
  • Advanced governance and compliance
  • Dedicated FinOps teams and tools

Cost Optimization Automation

AWS Lambda for Auto-Stopping Resources

import boto3
from datetime import datetime

def lambda_handler(event, context):
    ec2 = boto3.client('ec2')

    # Find development instances
    instances = ec2.describe_instances(
        Filters=[
            {'Name': 'tag:Environment', 'Values': ['development']},
            {'Name': 'instance-state-name', 'Values': ['running']}
        ]
    )

    instance_ids = []
    for reservation in instances['Reservations']:
        for instance in reservation['Instances']:
            instance_ids.append(instance['InstanceId'])

    # Stop instances outside business hours
    current_hour = datetime.now().hour
    if current_hour < 9 or current_hour > 17:  # Outside 9 AM - 5 PM
        if instance_ids:
            ec2.stop_instances(InstanceIds=instance_ids)
            print(f"Stopped {len(instance_ids)} development instances")

    return {'statusCode': 200}

Terraform for Cost-Optimized Infrastructure

# Cost-optimized EC2 instance
resource "aws_instance" "web" {
  ami           = data.aws_ami.amazon_linux.id
  instance_type = var.environment == "production" ? "t3.large" : "t3.micro"

  # Use spot instances for non-critical workloads
  instance_market_options {
    market_type = var.use_spot_instances ? "spot" : null
  }

  # Auto-scaling group for production
  dynamic "autoscaling_group" {
    for_each = var.environment == "production" ? [1] : []
    content {
      desired_capacity = 2
      min_size         = 1
      max_size         = 5
    }
  }

  tags = {
    Name        = "web-server"
    Environment = var.environment
    Team        = "engineering"
    Cost-Center = "web-platform"
  }
}
