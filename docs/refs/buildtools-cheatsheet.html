Build Tools Cheatsheet

Essential commands and configurations for popular build tools used in software development. This cheatsheet covers Make, Maven, Gradle, npm, and Yarn for efficient project building and dependency management.

Make

Make

Classic build automation tool using Makefiles

Basic Makefile Structure

Sample Makefile

# Variables
CC = gcc
CFLAGS = -Wall -O2
TARGET = myprogram
SOURCES = main.c utils.c
OBJECTS = $(SOURCES:.c=.o)

# Default target
all: $(TARGET)

# Link object files
$(TARGET): $(OBJECTS)
	$(CC) $(OBJECTS) -o $(TARGET)

# Compile source files
%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

# Clean build artifacts
clean:
	rm -f $(OBJECTS) $(TARGET)

# Install program
install: $(TARGET)
	install -d $(DESTDIR)/usr/local/bin
	install $(TARGET) $(DESTDIR)/usr/local/bin/

# Uninstall program
uninstall:
	rm -f $(DESTDIR)/usr/local/bin/$(TARGET)

# Run tests
test: $(TARGET)
	./$(TARGET) --test

.PHONY: all clean install uninstall test

Maven

Maven

Java project management and build tool

pom.xml Structure

Sample pom.xml

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0.0</version>
    <packaging>jar</packaging>

    <name>My Application</name>
    <description>Demo application</description>

    <properties>
        <maven.compiler.source>11</maven.compiler.source>
        <maven.compiler.target>11</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <dependencies>
        <dependency>
            <groupId>junit</groupId>
            <artifactId>junit</artifactId>
            <version>4.12</version>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <version>3.8.1</version>
                <configuration>
                    <source>11</source>
                    <target>11</target>
                </configuration>
            </plugin>
        </plugins>
    </build>
</project>

Gradle

Gradle

Modern build automation tool with Groovy/Kotlin DSL

build.gradle (Groovy)

Sample build.gradle

plugins {
    id 'java'
    id 'application'
}

group = 'com.example'
version = '1.0.0'
sourceCompatibility = '11'

repositories {
    mavenCentral()
}

dependencies {
    implementation 'com.google.guava:guava:31.1-jre'
    testImplementation 'junit:junit:4.13.2'
}

application {
    mainClass = 'com.example.App'
}

jar {
    manifest {
        attributes 'Main-Class': 'com.example.App'
    }
}

task customTask(type: Exec) {
    commandLine 'echo', 'Hello from Gradle!'
}

task helloWorld {
    doLast {
        println 'Hello World!'
    }
}

Build Tool Comparison

Tool Language Use Case Pros Cons
Make C/C++/Any General purpose Fast, flexible, platform independent Complex syntax, manual dependency management
Maven Java Java projects Convention over configuration, rich ecosystem XML configuration, slower than Gradle
Gradle Java/Scala/Groovy/Kotlin Modern JVM projects Flexible, fast, Groovy/Kotlin DSL Steep learning curve
npm JavaScript Node.js projects Largest package registry, simple Security issues, inconsistent lockfiles
Yarn JavaScript Node.js projects Faster, deterministic, workspaces Additional complexity

Best Practices

General Build Best Practices

Version Control
  • Never commit build artifacts
  • Include build tools in version control
  • Use semantic versioning
  • Tag releases in git
CI/CD Integration
  • Automate builds on every commit
  • Use build matrices for multiple environments
  • Cache dependencies for faster builds
  • Fail fast on build errors
Security
  • Scan dependencies for vulnerabilities
  • Use pinned versions in lockfiles
  • Don't commit secrets
  • Regular security audits
Performance
  • Use parallel builds when possible
  • Cache build artifacts
  • Minimize build steps
  • Profile build performance
