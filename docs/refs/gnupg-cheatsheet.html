GnuPG Cheatsheet

GnuPG (GNU Privacy Guard) is a complete and free implementation of the OpenPGP standard for encrypting and signing data. This cheatsheet covers key generation, management, encryption/decryption, and digital signatures.

Security Best Practices
  • Use strong passphrases (20+ characters, mixed case, numbers, symbols)
  • Regularly backup your private keys and revocation certificates
  • Never share your private keys or passphrases
  • Use subkeys for day-to-day operations
  • Keep your GnuPG installation updated
  • Verify key fingerprints before trusting keys

Key Generation

Generate New Key Pair

Interactive Key Generation

gpg --gen-key

# Follow the prompts:
# 1. Key type: RSA and RSA (default)
# 2. Key size: 4096 (recommended)
# 3. Expiration: 2y (2 years, or 0 for no expiration)
# 4. Name: Your Full Name
# 5. Email: your.email@example.com
# 6. Comment: (optional)
# 7. Passphrase: Choose a strong passphrase

Batch Key Generation

Unattended Key Generation

# Create batch configuration file
cat > keygen.conf << EOF
Key-Type: RSA
Key-Length: 4096
Subkey-Type: RSA
Subkey-Length: 4096
Name-Real: Your Name
Name-Comment: Key for secure communication
Name-Email: your.email@example.com
Expire-Date: 2y
Passphrase: your-secure-passphrase
EOF

# Generate key
gpg --batch --gen-key keygen.conf

# Clean up
rm keygen.conf

Configure Keyserver

GnuPG Configuration (~/.gnupg/gpg.conf)

# Keyserver configuration
keyserver hkps://keys.openpgp.org
keyserver-options no-honor-keyserver-url
keyserver-options include-revoked
keyserver-options verbose

# Default preferences
personal-cipher-preferences AES256 AES192 AES CAST5
personal-digest-preferences SHA512 SHA384 SHA256 SHA224
personal-compress-preferences ZLIB BZIP2 ZIP Uncompressed
default-preference-list AES256 AES192 AES CAST5 SHA512 SHA384 SHA256 SHA224 ZLIB BZIP2 ZIP Uncompressed

# Other settings
use-agent
keyid-format 0xlong
with-fingerprint
with-keygrip

# Auto-retrieve keys from keyserver
auto-key-retrieve

Thunderbird/Enigmail Setup

# Export public key for sharing
gpg --export --armor your.email@example.com > public.asc

# Import other's public key

# Encrypt email

gpg -se -r friend@example.com message.txt

Best Practices Summary

Key Management
  • Use RSA 4096-bit keys minimum
  • Create separate subkeys for signing/encryption
  • Set reasonable expiration dates
  • Backup keys and revocation certificates
  • Regular key rotation and maintenance
Security Practices
  • Never share private keys
  • Use strong, unique passphrases
  • Verify key fingerprints before trusting
  • Keep GnuPG updated
  • Use keyservers responsibly
Operational Best Practices
  • Use detached signatures for verification
  • Sign and encrypt sensitive communications
  • Regularly update key trust levels
  • Monitor key expiration dates
  • Document key management procedures
Integration Tips
  • Configure Git commit signing
  • Use with email clients for secure communication
  • Integrate with password managers
  • Automate encryption in scripts
  • Use with CI/CD pipelines
