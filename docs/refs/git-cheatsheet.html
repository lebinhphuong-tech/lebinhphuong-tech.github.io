Git Commands Cheatsheet

Essential Git commands for version control, branching, merging, and collaboration. This cheatsheet covers the most commonly used Git commands with practical examples.

Quick Start

Install Git: sudo apt install git (Ubuntu/Debian) or brew install git (macOS)
Configure: git config --global user.name "Your Name" and git config --global user.email "your.email@example.com"
Initialize: git init or clone: git clone https://github.com/user/repo.git

Basic Git Workflow

1
Initialize Repository: git init or git clone <url>
2
Make Changes: Edit files in your working directory
3
Stage Changes: git add <files>
4
Commit Changes: git commit -m "message"
5
Push Changes: git push origin <branch>

Repository Setup

Initialize & Clone

git init
Initialize a new Git repository
git clone <repository-url>
Clone an existing repository
git clone https://github.com/user/repo.git
git clone --depth 1 <url>
Shallow clone (only latest commit)
git remote add origin <url>
Add remote repository

Configuration

git config --global user.name "Name"
Set global username
git config --global user.email "email"
Set global email
git config --global core.editor "vim"
Set default editor
git config --list
Show all configuration

Basic Operations

Status & Changes

git status
Show working directory status
git diff
Show unstaged changes
git diff --staged
Show staged changes
git diff HEAD
Compare working directory with HEAD

Add & Commit

git add <file>
Stage specific file
git add index.html
git add .
Stage all changes in current directory
git commit -m "message"
Commit staged changes
git commit -m "Add user authentication"
git commit -am "message"
Add and commit tracked files

History & Logs

git log
Show commit history
git log --oneline
Show compact commit history
git log --graph --decorate
Show branch/merge graph
git blame <file>
Show who last modified each line

Branching & Merging

Branch Operations

git branch
List all branches
git branch <name>
Create new branch
git branch feature/login
git checkout <branch>
Switch to branch
git checkout feature/login
git checkout -b <branch>
Create and switch to new branch
git checkout -b feature/login

Merge & Rebase

git merge <branch>
Merge branch into current branch
git merge feature/login
git rebase <branch>
Rebase current branch onto another
git rebase main
git merge --abort
Abort a conflicted merge
git rebase --abort
Abort a conflicted rebase

Branch Management

git branch -d <branch>
Delete merged branch
git branch -D <branch>
Force delete branch
git branch -m <new-name>
Rename current branch
git branch --merged
List merged branches

Remote Operations

Push & Pull

git push origin <branch>
Push branch to remote
git push origin main
git push -u origin <branch>
Push and set upstream
git pull
Fetch and merge from remote
git pull --rebase
Pull with rebase instead of merge

Fetch & Remote

git fetch
Download objects from remote
git fetch origin
Fetch from specific remote
git remote -v
List remote repositories
git remote set-url origin <url>
Change remote URL

Undo & Fix

Working Directory

git checkout -- <file>
Discard changes in working directory
git reset HEAD <file>
Unstage file (keep changes)
git reset --hard HEAD
Reset to HEAD (discard all changes)
git clean -fd
Remove untracked files and directories

Commit History

git commit --amend
Modify last commit message
git reset --soft HEAD~1
Undo last commit (keep changes staged)
git reset --hard HEAD~1
Undo last commit (discard changes)
git revert <commit>
Create new commit that undoes changes

Advanced Reset

git reset --mixed HEAD~2
Reset to 2 commits ago (default)
git reflog
Show reference log (recovery)
git reset --hard <commit-hash>
Reset to specific commit

Stashing

Basic Stash

git stash
Stash working directory changes
git stash save "message"
Stash with custom message
git stash save "WIP: user authentication"
git stash list
List all stashes
git stash show
Show changes in latest stash

Apply & Drop

git stash apply
Apply latest stash (keep in list)
git stash pop
Apply and remove latest stash
git stash drop stash@{1}
Remove specific stash
git stash clear
Remove all stashes

Tags

Create Tags

git tag <tag-name>
Create lightweight tag
git tag v1.0.0
git tag -a <tag> -m "message"
Create annotated tag
git tag -a v1.0.0 -m "Release version 1.0.0"
git tag -l "v1.*"
List tags matching pattern
git show <tag>
Show tag information

Push Tags

git push origin <tag>
Push specific tag
git push origin --tags
Push all tags
git tag -d <tag>
Delete local tag
git push origin :refs/tags/<tag>
Delete remote tag

Common Git Workflows

Feature Branch Workflow

# 1. Create feature branch
git checkout -b feature/new-feature

# 2. Make changes and commit
git add .
git commit -m "Implement new feature"

# 3. Push feature branch
git push -u origin feature/new-feature

# 4. Create pull request/merge request
# (via GitHub/GitLab UI)

# 5. Merge to main (after review)
git checkout main
git pull origin main
git merge feature/new-feature

# 6. Clean up
git branch -d feature/new-feature
git push origin --delete feature/new-feature

Hotfix Workflow

# 1. Create hotfix branch from main
git checkout main
git pull origin main
git checkout -b hotfix/critical-bug

# 2. Fix the bug
# Edit files...
git add .
git commit -m "Fix critical bug in authentication"

# 3. Push and create pull request
git push -u origin hotfix/critical-bug

# 4. Merge to main and deploy
git checkout main
git merge hotfix/critical-bug
git push origin main

# 5. Also merge to develop if exists
git checkout develop
git merge hotfix/critical-bug
git push origin develop

Git Rebase Workflow

# Keep feature branch up to date
git checkout feature/my-feature
git fetch origin
git rebase origin/main

# Interactive rebase to clean commits
git rebase -i HEAD~3  # Last 3 commits

# Commands in interactive rebase:
# pick - keep commit
# squash - merge with previous
# reword - change message
# drop - remove commit

Git Best Practices

Do's
  • Write clear, descriptive commit messages
  • Use branches for new features and bug fixes
  • Pull frequently to stay up to date
  • Review code before merging
  • Use .gitignore for sensitive/unnecessary files
  • Tag releases for version tracking
  • Use meaningful branch names
Don'ts
  • Don't commit directly to main branch
  • Don't use generic messages like "fix" or "update"
  • Don't force push to shared branches
  • Don't commit large binary files
  • Don't rewrite public commit history
  • Don't mix unrelated changes in one commit
  • Don't ignore merge conflicts

Git Configuration Tips

Useful Aliases

# Add these to ~/.gitconfig
[alias]
  co = checkout
  br = branch
  ci = commit
  st = status
  unstage = reset HEAD --
  last = log -1 HEAD
  visual = !gitk
  lg = log --oneline --graph --decorate
  recent = log --oneline -10

# Usage
git co feature-branch    # instead of git checkout
git br -a               # instead of git branch -a
git lg                  # nice log graph
