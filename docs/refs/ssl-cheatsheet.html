SSL/TLS Cheatsheet

SSL/TLS (Secure Sockets Layer/Transport Layer Security) provides encryption and authentication for network communications. This cheatsheet covers certificate generation, management, and common SSL/TLS operations using OpenSSL.

Security Best Practices
  • Always use strong key sizes (2048-bit minimum, prefer 4096-bit)
  • Use certificate pinning for sensitive communications
  • Keep private keys secure and never share them
  • Regularly rotate certificates before expiration
  • Use Certificate Transparency for public certificates
  • Implement HSTS headers for web applications

Certificate Authorities

Creating a CA

Root CA Creation

# 1. Generate CA private key
openssl genrsa -out ca.key 4096

# 2. Create CA certificate
openssl req -x509 -new -key ca.key -sha256 -days 3650 -out ca.crt \
  -subj "/C=US/ST=State/L=City/O=MyCA/CN=My Root CA"

# 3. Create CA database files
touch index.txt
echo '01' > serial.txt

# 4. Create CA configuration
cat > ca.conf << EOF
[ ca ]
default_ca = CA_default

[ CA_default ]
dir = .
database = index.txt
serial = serial.txt
certificate = ca.crt
private_key = ca.key
default_days = 365
default_md = sha256
policy = policy_anything

[ policy_anything ]
countryName = optional
stateOrProvinceName = optional
localityName = optional
organizationName = optional
organizationalUnitName = optional
commonName = supplied
emailAddress = optional

[ req ]
distinguished_name = req_distinguished_name
req_extensions = v3_req

[ req_distinguished_name ]
commonName = Common Name

[ v3_req ]
basicConstraints = CA:FALSE
keyUsage = nonRepudiation, digitalSignature, keyEncipherment
extendedKeyUsage = serverAuth, clientAuth
EOF

Certificate Management Scripts

Certificate Expiry Monitoring

Bash Script for Certificate Monitoring

#!/bin/bash

# Certificate expiry checker
check_cert_expiry() {
    local cert_file="$1"
    local days_warning="${2:-30}"

    if [ ! -f "$cert_file" ]; then
        echo "Certificate file not found: $cert_file"
        return 1
    fi

    local end_date=$(openssl x509 -in "$cert_file" -noout -enddate | cut -d= -f2)
    local end_epoch=$(date -d "$end_date" +%s)
    local now_epoch=$(date +%s)
    local days_left=$(( ($end_epoch - $now_epoch) / 86400 ))

    echo "Certificate: $cert_file"
    echo "Expiry date: $end_date"
    echo "Days remaining: $days_left"

    if [ $days_left -le $days_warning ]; then
        echo "WARNING: Certificate expires in $days_left days!"
        return 1
    else
        echo "Certificate is valid"
        return 0
    fi
}

# Usage
check_cert_expiry "/etc/ssl/certs/example.crt" 30

Automated Certificate Generation

Automated Self-Signed Certificate Script

#!/bin/bash

# Generate self-signed certificate
generate_self_signed_cert() {
    local domain="$1"
    local days="${2:-365}"
    local key_file="${domain}.key"
    local cert_file="${domain}.crt"

    # Generate private key
    openssl genrsa -out "$key_file" 2048

    # Generate certificate
    openssl req -new -x509 -key "$key_file" -out "$cert_file" -days "$days" \
        -subj "/C=US/ST=State/L=City/O=Organization/CN=${domain}" \
        -addext "subjectAltName = DNS:${domain},DNS:www.${domain}"

    echo "Generated certificate: $cert_file"
    echo "Generated key: $key_file"
}

# Usage
generate_self_signed_cert "example.com" 365

SSL/TLS Best Practices Summary

Key Management
  • Use RSA 2048-bit or ECDSA P-256 minimum
  • Store private keys securely (encrypted)
  • Regular key rotation (annually)
  • Use hardware security modules for production
Certificate Practices
  • Use Subject Alternative Names (SAN)
  • Monitor certificate expiry (automated)
  • Implement certificate pinning
  • Use Certificate Transparency
TLS Configuration
  • Disable SSL/TLS 1.0 and 1.1
  • Use strong cipher suites only
  • Enable HSTS headers
  • Implement proper session resumption
Monitoring & Compliance
  • Regular vulnerability scanning
  • Certificate transparency monitoring
  • Compliance with PCI DSS, HIPAA, etc.
  • Automated alerting for issues
