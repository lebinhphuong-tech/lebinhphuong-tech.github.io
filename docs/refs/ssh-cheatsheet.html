SSH Commands Cheatsheet

SSH (Secure Shell) is a cryptographic network protocol for operating network services securely over an unsecured network. This cheatsheet covers essential SSH commands, key management, tunneling, and security best practices.

Security Best Practices
  • Never use password authentication in production
  • Use strong, unique SSH keys for each server
  • Disable root login and password authentication
  • Use fail2ban or similar tools to prevent brute force attacks
  • Regularly rotate SSH keys and update authorized_keys

SSH Key Management

Key Generation

RSA Keys (Default)

ssh-keygen -t rsa -b 4096
Generate RSA key pair (4096 bits)
ssh-keygen -t rsa -b 4096 -C "user@domain.com"
Generate RSA key with comment

Ed25519 Keys (Modern)

ssh-keygen -t ed25519
Generate Ed25519 key pair (recommended)
ssh-keygen -t ed25519 -C "user@domain.com"
Generate Ed25519 key with comment

Key Distribution

Copy Public Key

ssh-copy-id user@hostname
Copy public key to remote host
ssh-copy-id -i ~/.ssh/id_rsa.pub user@hostname
Copy specific key to remote host

Manual Setup

cat ~/.ssh/id_rsa.pub | ssh user@hostname "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
Manually append public key
ssh user@hostname "chmod 600 ~/.ssh/authorized_keys"
Set correct permissions on remote host

Key Management

Key Information

ssh-keygen -l -f ~/.ssh/id_rsa.pub
Show key fingerprint and type
ssh-keygen -l -v -f ~/.ssh/id_rsa.pub
Show detailed key information

Key Conversion

ssh-keygen -p -f ~/.ssh/id_rsa
Change private key passphrase
ssh-keygen -y -f ~/.ssh/id_rsa
Extract public key from private key

SSH Tunneling & Port Forwarding

Local Port Forwarding

Basic Local Forwarding

ssh -L 8080:localhost:80 user@hostname
Forward local port 8080 to remote port 80
ssh -L 3306:db.example.com:3306 user@jumpbox
Access remote database through jumpbox

Dynamic Port Forwarding

ssh -D 8080 user@hostname
Create SOCKS proxy on local port 8080
ssh -D 1080 -N user@hostname
SOCKS proxy without shell (-N flag)

Remote Port Forwarding

Basic Remote Forwarding

ssh -R 8080:localhost:80 user@hostname
Forward remote port 8080 to local port 80
ssh -R 2222:localhost:22 user@jumpbox
Allow external access to local SSH

Reverse Tunnel

ssh -R 0.0.0.0:8080:localhost:80 user@server
Bind to all interfaces (dangerous)
ssh -R 8080:localhost:80 -N user@server
Reverse tunnel without shell

SSH Configuration & Automation

SSH Config File (~/.ssh/config)

SSH Config Examples

# Basic host configuration
Host myserver
    HostName server.example.com
    User myuser
    Port 22
    IdentityFile ~/.ssh/id_rsa

# Jump host configuration
Host internal-server
    HostName 192.168.1.100
    User admin
    ProxyJump jumpbox.example.com

# Multiple identity files
Host github.com
    User git
    IdentityFile ~/.ssh/id_rsa_github
    IdentityFile ~/.ssh/id_ed25519_github

# Connection options
Host *
    ServerAliveInterval 60
    ServerAliveCountMax 3
    StrictHostKeyChecking ask
    UserKnownHostsFile ~/.ssh/known_hosts

# Disable password authentication
Host production
    PasswordAuthentication no
    PubkeyAuthentication yes

SSH Server Configuration

sshd_config (/etc/ssh/sshd_config)

Security Hardening

# Disable root login
PermitRootLogin no

# Disable password authentication
PasswordAuthentication no
ChallengeResponseAuthentication no

# Use only strong ciphers
Ciphers aes256-gcm@openssh.com,chacha20-poly1305@openssh.com
MACs hmac-sha2-512-etm@openssh.com,hmac-sha2-256-etm@openssh.com

# Limit authentication attempts
MaxAuthTries 3
LoginGraceTime 30

# Use strong key exchange algorithms
KexAlgorithms curve25519-sha256@libssh.org,diffie-hellman-group16-sha512

# Enable X11 forwarding (if needed)
X11Forwarding yes
X11UseLocalhost yes

# Keep connections alive
ClientAliveInterval 60
ClientAliveCountMax 3

# Log more information
LogLevel VERBOSE

Advanced SSH Techniques

SCP & SFTP

SCP Commands

scp file.txt user@hostname:/path/
Copy file to remote host
scp -r directory/ user@hostname:/path/
Copy directory recursively
scp user@hostname:/path/file.txt .
Copy file from remote host

SFTP Commands

sftp user@hostname
Start SFTP interactive session
sftp -b batchfile user@hostname
Execute SFTP commands from file

SSH Escape Sequences

Useful SSH Escape Sequences
  • ~. - Terminate connection
  • ~C - Open command line (for port forwarding)
  • ~R - Request rekeying
  • ~V - Decrease verbosity
  • ~v - Increase verbosity
  • ~~ - Send literal tilde

SSH Bastion/Jump Host

Jump Host Configuration
# SSH config for jump host
Host jumpbox
    HostName bastion.example.com
    User jumpuser
    IdentityFile ~/.ssh/jump_key

Host internal-server
    HostName 10.0.1.100
    User admin
    ProxyCommand ssh -W %h:%p jumpbox
    IdentityFile ~/.ssh/internal_key

# Alternative using ProxyJump
Host internal-server
    HostName 10.0.1.100
    User admin
    ProxyJump jumpuser@bastion.example.com
    IdentityFile ~/.ssh/internal_key

SSH Troubleshooting

Debugging Connections

ssh -vvv user@hostname
Maximum verbosity for troubleshooting
ssh -o ConnectTimeout=10 user@hostname
Set connection timeout
ssh -o StrictHostKeyChecking=no user@hostname
Skip host key verification (dangerous)

Key Issues

ssh-keygen -R hostname
Remove host from known_hosts
ssh-keyscan hostname >> ~/.ssh/known_hosts
Add host key to known_hosts

SSH Performance Optimization

Performance Tuning

# Client-side optimizations
Host *
    # Connection multiplexing
    ControlMaster auto
    ControlPath ~/.ssh/master-%r@%h:%p
    ControlPersist 10m

    # Compression
    Compression yes
    CompressionLevel 6

    # Connection reuse
    ServerAliveInterval 60
    ServerAliveCountMax 10

    # Faster ciphers (if security allows)
    Ciphers aes128-gcm@openssh.com,aes256-gcm@openssh.com

# Server-side optimizations
# In /etc/ssh/sshd_config
UseDNS no
GSSAPIAuthentication no
UsePAM yes

# TCP optimizations
TCPKeepAlive yes
ClientAliveInterval 60
ClientAliveCountMax 3
