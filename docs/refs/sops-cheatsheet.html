SOPS Cheatsheet

SOPS (Secrets OPerationS) is an editor of encrypted files that supports YAML, JSON, ENV, INI and BINARY formats. It encrypts/decrypts secrets using AWS KMS, GCP KMS, Azure Key Vault, age, and PGP.

Security Best Practices
  • Use strong encryption keys (KMS, age, PGP)
  • Never commit decrypted secrets to version control
  • Rotate encryption keys regularly
  • Use separate keys for different environments
  • Implement proper key management and access controls

Installation

Linux/macOS

curl -LO https://github.com/mozilla/sops/releases/latest/download/sops-linux
Download SOPS binary for Linux
chmod +x sops-linux && sudo mv sops-linux /usr/local/bin/sops
Make executable and move to PATH

macOS with Homebrew

brew install sops
Install via Homebrew
sops --version
Verify installation

Basic Operations

Encrypting Files

Basic Encryption

sops -e secrets.yaml > secrets.enc.yaml
Encrypt YAML file
sops -e --input-type json secrets.json > secrets.enc.json
Encrypt JSON file with explicit type
sops -e secrets.env > secrets.enc.env
Encrypt environment file

In-place Encryption

sops -e -i secrets.yaml
Encrypt file in-place
sops --encrypt --in-place secrets.json
Long form in-place encryption

Decrypting Files

Basic Decryption

sops -d secrets.enc.yaml
Decrypt YAML file to stdout
sops -d secrets.enc.yaml > secrets.decrypted.yaml
Decrypt and save to file
sops --decrypt secrets.enc.json
Long form decryption

View Encrypted Files

sops secrets.enc.yaml
View encrypted file (decrypts automatically)
cat secrets.enc.yaml | sops -d -
Decrypt from stdin

Key Management

AWS KMS

Using AWS KMS for Encryption

# Create KMS key
aws kms create-key --description "SOPS encryption key"

# Get key ARN (replace with actual ARN)
KEY_ARN="arn:aws:kms:us-east-1:123456789012:key/12345678-1234-1234-1234-123456789012"

# Encrypt with KMS
sops -e --kms $KEY_ARN secrets.yaml > secrets.enc.yaml

# For multiple KMS keys (production + development)
PROD_KEY="arn:aws:kms:us-east-1:123456789012:key/prod-key"
DEV_KEY="arn:aws:kms:us-east-1:123456789012:key/dev-key"

sops -e --kms $PROD_KEY --kms $DEV_KEY secrets.yaml > secrets.enc.yaml

GCP KMS

Using Google Cloud KMS

# Create KMS key
gcloud kms keyrings create sops --location global
gcloud kms keys create sops-key --location global --keyring sops --purpose encryption

# Get key resource name
KEY_RESOURCE="projects/myproject/locations/global/keyRings/sops/cryptoKeys/sops-key"

# Encrypt with GCP KMS
sops -e --gcp-kms $KEY_RESOURCE secrets.yaml > secrets.enc.yaml

Azure Key Vault

Using Azure Key Vault

# Create Key Vault and key
az keyvault create --name mykeyvault --resource-group mygroup --location eastus
az keyvault key create --vault-name mykeyvault --name sops-key --protection software

# Get key URL
KEY_URL="https://mykeyvault.vault.azure.net/keys/sops-key"

# Encrypt with Azure Key Vault
sops -e --azure-kv $KEY_URL secrets.yaml > secrets.enc.yaml

Age Encryption

Using Age for Encryption

# Generate age key pair
age-keygen -o key.txt

# Key file content (example):
# # created: 2023-01-01T00:00:00+00:00
# # public key: age1xyz...
# AGE-SECRET-KEY-1XYZ...

# Extract public key
PUBLIC_KEY=$(age-keygen -y key.txt)

# Encrypt with age
sops -e --age $PUBLIC_KEY secrets.yaml > secrets.enc.yaml

# For multiple recipients
sops -e --age $PUBLIC_KEY1 --age $PUBLIC_KEY2 secrets.yaml > secrets.enc.yaml

PGP/GPG

Using PGP/GPG Keys

# Generate GPG key (or use existing)
gpg --gen-key

# List keys to get key ID
gpg --list-keys

# Get key fingerprint
KEY_FP=$(gpg --list-keys --with-colons user@example.com | grep '^fpr' | cut -d: -f10)

# Encrypt with PGP
sops -e --pgp $KEY_FP secrets.yaml > secrets.enc.yaml

# For multiple PGP keys
sops -e --pgp $KEY_FP1 --pgp $KEY_FP2 secrets.yaml > secrets.enc.yaml

Configuration Files

SOPS Configuration (.sops.yaml)

Sample .sops.yaml Configuration

creation_rules:
  # Encrypt everything by default
  - path_regex: .*
    kms: 'arn:aws:kms:us-east-1:123456789012:key/12345678-1234-1234-1234-123456789012'
    pgp: '1234567890ABCDEF1234567890ABCDEF12345678'

  # Specific rules for different environments
  - path_regex: .*prod.*
    kms: 'arn:aws:kms:us-east-1:123456789012:key/prod-key-arn'
    gcp_kms: 'projects/myproject/locations/global/keyRings/prod/cryptoKeys/prod-key'

  - path_regex: .*dev.*
    age: 'age1xyz...'

  # Exclude certain files or keys
  - path_regex: .*test.*
    encrypted_regex: '^(password|secret)$'

# Key groups for multiple encryption keys
key_groups:
  - kms:
      - arn: arn:aws:kms:us-east-1:123456789012:key/key1
      - arn: arn:aws:kms:us-west-2:123456789012:key/key2
    pgp:
      - 1234567890ABCDEF1234567890ABCDEF12345678
      - 9876543210FEDCBA9876543210FEDCBA98765432

Selective Encryption

Encrypt Specific Keys

sops -e --encrypted-regex '^(password|secret)$' secrets.yaml
Encrypt only password and secret keys
sops -e --unencrypted-regex '^public_' secrets.yaml
Leave public_ prefixed keys unencrypted

Key Patterns

sops -e --encrypted-regex 'db.*' secrets.yaml
Encrypt all keys starting with 'db'
sops -e --encrypted-regex '.*_password$' secrets.yaml
Encrypt all keys ending with '_password'

Integration Examples

Kubernetes Integration

Using SOPS with Kubernetes

# Encrypt secrets file
sops -e secrets.yaml > secrets.enc.yaml

# View decrypted content
sops -d secrets.enc.yaml

# Use in Kubernetes with KSOPS or similar
kubectl apply -f <(sops -d secrets.enc.yaml)

# Or use kustomize with sops
# Create kustomization.yaml
cat > kustomization.yaml << EOF
secretGenerator:
- name: mysecret
  literals:
  - username=admin
  - password=secret
  generators:
  - sops:
      file: secrets.enc.yaml
EOF

kubectl apply -k .

CI/CD Integration

GitHub Actions with SOPS

# .github/workflows/deploy.yml
name: Deploy
on: push

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3

    - name: Setup SOPS
      uses: mdgreenwald/mozilla-sops-action@v1

    - name: Decrypt secrets
      run: sops -d secrets.enc.yaml > secrets.yaml

    - name: Deploy
      run: ./deploy.sh
      env:
        SECRET_KEY: ${{ secrets.SOPS_AGE_KEY }}

Terraform Integration

Using SOPS with Terraform

# Encrypt Terraform variables
sops -e terraform.tfvars > terraform.enc.tfvars

# Decrypt for terraform apply
sops -d terraform.enc.tfvars > terraform.tfvars

# Or use terraform-sops provider
terraform {
  required_providers {
    sops = {
      source = "carlpett/sops"
      version = "~> 0.7"
    }
  }
}

data "sops_file" "secrets" {
  source_file = "secrets.enc.yaml"
}

output "database_password" {
  value = data.sops_file.secrets.data["database.password"]
  sensitive = true
}

Best Practices

Key Management
  • Use separate keys for each environment
  • Regular key rotation (quarterly/annually)
  • Backup encryption keys securely
  • Implement proper access controls for keys
  • Use KMS over local keys when possible
File Organization
  • Use consistent file naming (.enc.yaml)
  • Keep .sops.yaml in repository root
  • Separate secrets by environment
  • Document decryption requirements
  • Exclude encrypted files from git if needed
CI/CD Integration
  • Decrypt secrets at runtime, not build time
  • Use environment-specific keys
  • Implement secret scanning in CI
  • Automate key rotation in pipelines
  • Monitor secret access and usage
Security Practices
  • Never commit decrypted secrets
  • Use strong encryption keys
  • Implement audit logging
  • Regular security assessments
  • Follow principle of least privilege
