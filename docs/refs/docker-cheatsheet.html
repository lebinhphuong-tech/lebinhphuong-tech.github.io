Docker Commands Cheatsheet

Essential Docker commands for container management, image building, networking, and volume operations. This cheatsheet covers the most commonly used Docker commands with practical examples.

Quick Start

Install Docker: curl -fsSL https://get.docker.com | sh
Start Docker service: sudo systemctl start docker
Add user to docker group: sudo usermod -aG docker $USER

Container Management

Run Containers

docker run [OPTIONS] IMAGE [COMMAND]
Run a container from an image
docker run -d -p 8080:80 nginx
docker run -it ubuntu bash
docker run --name container_name
Assign a name to the container
docker run -d
Run container in background (detached)
docker run -p host_port:container_port
Publish container port to host

List & Inspect

docker ps
List running containers
docker ps -a
List all containers (including stopped)
docker inspect container_name
Show detailed container information
docker inspect my_container | grep -A 10 "NetworkSettings"
docker stats
Show resource usage statistics

Start & Stop

docker start container_name
Start a stopped container
docker stop container_name
Stop a running container gracefully
docker kill container_name
Force stop a container
docker restart container_name
Restart a container

Execute & Logs

docker exec -it container_name command
Execute command in running container
docker exec -it my_container bash
docker logs container_name
View container logs
docker logs -f my_container
docker logs --tail 100 container_name
Show last 100 log lines
docker attach container_name
Attach to container's stdin/stdout

Image Management

Build Images

docker build -t image_name:tag .
Build image from Dockerfile in current directory
docker build -t myapp:1.0 .
docker build --no-cache
Build without using cache
docker build -f Dockerfile.custom
Specify custom Dockerfile name
docker build --build-arg key=value
Pass build-time variables

List & Remove

docker images
List all local images
docker rmi image_name:tag
Remove an image
docker rmi nginx:latest
docker image prune
Remove unused images
docker image prune -a
Remove all unused images

Registry Operations

docker pull image_name:tag
Download image from registry
docker pull nginx:alpine
docker push image_name:tag
Upload image to registry
docker login registry_url
Login to registry
docker login ghcr.io
docker tag source_image target_image
Tag an image
docker tag myapp:latest ghcr.io/user/myapp:v1.0

Volumes & Networks

Volume Management

docker volume create volume_name
Create a named volume
docker volume ls
List all volumes
docker volume rm volume_name
Remove a volume
docker volume prune
Remove unused volumes
docker run -v volume_name:/path/in/container
Mount named volume
docker run -v /host/path:/container/path
Bind mount host directory

Network Management

docker network create network_name
Create a user-defined network
docker network ls
List all networks
docker network connect network_name container
Connect container to network
docker network disconnect network_name container
Disconnect container from network
docker run --network network_name
Run container on specific network
docker network prune
Remove unused networks

Docker Compose

Basic Operations

docker-compose up
Create and start containers
docker-compose up -d
Start in detached mode
docker-compose down
Stop and remove containers
docker-compose logs
View service logs
docker-compose logs -f web

Service Management

docker-compose ps
List services status
docker-compose build
Build or rebuild services
docker-compose restart service_name
Restart specific service
docker-compose exec service_name command
Execute command in service container
docker-compose exec web bash

Configuration

docker-compose config
Validate and view compose file
docker-compose pull
Pull service images
docker-compose scale service=num
Scale services (deprecated in v3)
docker-compose --file custom-compose.yml
Use custom compose file

System & Cleanup

System Information

docker version
Show Docker version information
docker info
Display system-wide information
docker system df
Show Docker disk usage
docker system prune
Remove unused data (containers, networks, images)

Cleanup Commands

docker container prune
Remove stopped containers
docker image prune
Remove unused images
docker volume prune
Remove unused volumes
docker network prune
Remove unused networks
docker system prune -a
Remove all unused Docker objects

Common Docker Patterns

Development Environment

# Run development container with volume mounting
docker run -d --name dev-env \
  -v $(pwd):/app \
  -p 3000:3000 \
  -e NODE_ENV=development \
  node:16 \
  npm start

# Access container shell
docker exec -it dev-env bash

Multi-stage Build

# Dockerfile with multi-stage build
# Build stage
FROM node:16 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

# Production stage
FROM node:16-slim
WORKDIR /app
COPY --from=builder /app/node_modules ./node_modules
COPY . .

EXPOSE 3000
CMD ["npm", "start"]

Database with Persistent Storage

# Run PostgreSQL with persistent volume
docker run -d --name postgres \
  -e POSTGRES_PASSWORD=mypass \
  -e POSTGRES_DB=myapp \
  -v postgres_data:/var/lib/postgresql/data \
  -p 5432:5432 \
  postgres:13

# Backup database
docker exec postgres pg_dump -U postgres myapp > backup.sql

Docker Compose for Full Stack

# docker-compose.yml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "3000:3000"
    depends_on:
      - db
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/app

  db:
    image: postgres:13
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: app
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:

Docker Best Practices

Do's
  • Use specific image tags (not :latest)
  • Keep containers stateless
  • Use multi-stage builds to reduce image size
  • Run containers as non-root user
  • Use .dockerignore file
  • Label your images appropriately
  • Keep base images minimal (use alpine variants)
Don'ts
  • Don't store data in containers (use volumes)
  • Don't run multiple processes in one container
  • Don't use root user in containers
  • Don't commit secrets to images
  • Don't use :latest tag in production
  • Don't expose unnecessary ports
  • Don't build large images without cleanup
