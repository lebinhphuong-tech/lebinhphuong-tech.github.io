Cybersecurity Core Concepts

Understanding cybersecurity fundamentals is essential for building secure systems and protecting against cyber threats. This guide covers the core concepts, principles, and terminology that form the foundation of information security.

Cryptography Fundamentals

Symmetric Encryption

Single Key for Encryption and Decryption

Symmetric encryption uses the same key for both encryption and decryption operations.

Common Algorithms:

  • AES (Advanced Encryption Standard) - 128, 192, 256-bit keys
  • DES (Data Encryption Standard) - Legacy, 56-bit keys
  • 3DES - Triple DES for enhanced security
  • ChaCha20 - Stream cipher, fast on mobile devices

Use Cases:

  • Encrypting data at rest (databases, files)
  • Secure communication within trusted environments
  • VPN connections
  • Full disk encryption

Asymmetric Encryption

Public/Private Key Pairs

Asymmetric encryption uses different keys for encryption and decryption, enabling secure key exchange.

Common Algorithms:

  • RSA - 2048-bit or higher key sizes
  • ECC (Elliptic Curve) - Smaller keys, equivalent security
  • Ed25519 - Modern ECC variant

Use Cases:

  • Digital signatures and certificates
  • Key exchange (TLS/SSL)
  • Secure email (PGP)
  • Blockchain transactions

Hashing Functions

One-Way Cryptographic Functions

Hash functions convert input data into fixed-size output (digest) that cannot be reversed.

Common Algorithms:

  • SHA-256 - Widely used, 256-bit output
  • SHA-3 - Next-generation hash function
  • BLAKE3 - Fast, modern hash function
  • bcrypt/scrypt/PBKDF2 - Password hashing

Use Cases:

  • Password storage (with salt)
  • File integrity verification
  • Digital signatures
  • Blockchain proof-of-work

Digital Certificates & PKI

Public Key Infrastructure

PKI provides the framework for managing digital certificates and public-key encryption.

PKI Components:

  • Certificate Authority (CA) - Issues and validates certificates
  • Registration Authority (RA) - Verifies certificate requests
  • Certificate Repository - Stores certificates and CRLs
  • Certificate Revocation List (CRL) - List of revoked certificates
  • Online Certificate Status Protocol (OCSP) - Real-time certificate validation

Network Security

Defense in Depth

Multiple Layers of Security

Implementing multiple security controls so that if one layer fails, others provide protection.

Security Layers:

  1. Physical Security - Secure facilities and hardware
  2. Network Security - Firewalls, IDS/IPS, segmentation
  3. Host Security - Endpoint protection, hardening
  4. Application Security - Secure coding, input validation
  5. Data Security - Encryption, access controls

Network Segmentation

Dividing Networks into Segments

Network segmentation limits the spread of breaches by dividing networks into isolated segments.

Segmentation Types:

  • VLAN Segmentation - Virtual LAN isolation
  • Subnet Segmentation - IP-based network division
  • Micro-Segmentation - East-west traffic control
  • Zero Trust Segmentation - Identity-based access control

Benefits:

  • Contain breach impact
  • Implement least privilege
  • Simplify compliance
  • Improve performance monitoring

Vulnerability Management

CVSS Scoring System

Common Vulnerability Scoring System

CVSS provides a standardized way to assess and communicate vulnerability severity.

CVSS Metrics:

  • Base Score (0-10) - Fundamental vulnerability characteristics
  • Temporal Score - Current threat level and exploitability
  • Environmental Score - Impact on specific environment

Severity Levels:

  • 0.0 - None - No vulnerability
  • 0.1-3.9 - Low - Minimal impact
  • 4.0-6.9 - Medium - Moderate impact
  • 7.0-8.9 - High - Significant impact
  • 9.0-10.0 - Critical - Maximum impact

Incident Response

Incident Response Lifecycle

NIST Incident Response Framework

Structured approach to handling security incidents effectively.

Phases:

  1. Preparation - Build capabilities and resources
  2. Identification - Detect and assess incidents
  3. Containment - Limit incident scope and impact
  4. Eradication - Remove root cause and threats
  5. Recovery - Restore systems and return to normal
  6. Lessons Learned - Review and improve processes

Risk Management

Risk Assessment Process

Systematic Risk Evaluation

Risk assessment helps organizations prioritize security investments and controls.

Risk Formula:

Risk = Threat × Vulnerability × Impact

Risk Assessment Steps:

  1. Identify Assets - What needs protection?
  2. Identify Threats - What could go wrong?
  3. Identify Vulnerabilities - Weaknesses that could be exploited
  4. Assess Impact - Consequences if threat is realized
  5. Determine Risk Level - Likelihood × impact
  6. Implement Controls - Mitigate high-priority risks

Risk Treatment Strategies

Strategy Description Pros Cons
Risk Avoidance Eliminate the risk by not doing the activity Complete protection May miss opportunities
Risk Reduction Implement controls to reduce likelihood or impact Balanced approach Residual risk remains
Risk Transfer Transfer risk to third party (insurance, outsourcing) Financial protection Cost implications
Risk Acceptance Acknowledge and accept the risk No implementation cost Potential for loss

Security Testing

Testing Methodologies

Comprehensive Security Testing

Testing Types:

  • Static Application Security Testing (SAST) - Code analysis without execution
  • Dynamic Application Security Testing (DAST) - Runtime vulnerability scanning
  • Interactive Application Security Testing (IAST) - Runtime analysis during testing
  • Software Composition Analysis (SCA) - Third-party component vulnerabilities
  • Penetration Testing - Simulated attacks by ethical hackers
  • Red Team Exercises - Full-scale attack simulations

Security Testing in CI/CD

DevSecOps Integration

Integrating security testing into the development pipeline for continuous security.

Pipeline Security Gates:

  • Pre-commit - Static analysis, secret scanning
  • Build - SAST, dependency scanning
  • Test - DAST, API security testing
  • Deploy - Infrastructure scanning, compliance checks
  • Runtime - Continuous monitoring and alerting
✅ Key Security Concepts Summary:
  • CIA Triad - Confidentiality, Integrity, Availability
  • Authentication - Verify identity (something you know/have/are)
  • Authorization - Control access (RBAC, ABAC, PBAC)
  • Encryption - Symmetric (AES) vs Asymmetric (RSA) cryptography
  • Defense in Depth - Multiple security layers
  • Zero Trust - Never trust, always verify
  • Risk Management - Identify, assess, treat, monitor risks
  • Incident Response - Prepare, identify, contain, eradicate, recover
Common Security Misconceptions
  • "We haven't been hacked, so we're secure" - Many breaches go undetected
  • "Firewalls make us secure" - Defense in depth required
  • "Encryption solves everything" - Key management is critical
  • "Security is IT's job" - Everyone shares responsibility
  • "Compliance equals security" - Compliance is minimum requirement
