Ansible Documentation

Configuration Management & Automation - Ansible is an open-source automation tool that simplifies configuration management, application deployment, and task automation.

What is Ansible?

Ansible is an open-source software provisioning, configuration management, and application-deployment tool. It runs on many Unix-like systems, and can configure both Unix-like systems as well as Microsoft Windows. It includes its own declarative language to describe system configuration.

Key Features

  • 🚀 Agentless - No agents to install on managed nodes
  • 📝 Declarative - Describe desired state, not steps to achieve it
  • 🔧 Idempotent - Safe to run multiple times
  • 📚 Human-readable - YAML-based playbooks are easy to understand
  • 🔌 Extensible - Thousands of modules for various tasks
  • 🏗️ Infrastructure as Code - Version control your infrastructure

Architecture Overview

Control Node

Ansible Core

The main engine that executes playbooks and communicates with managed nodes.

Inventory

Defines the hosts and groups of hosts that Ansible will manage.

Playbooks

YAML files containing a series of tasks to be executed on managed nodes.

Managed Nodes

Target Systems

Remote systems that Ansible manages. Only require SSH and Python.

Modules

Reusable units of code that Ansible executes on managed nodes.

Installation

Install Ansible on Control Node

Linux (Ubuntu/Debian)

# Update package index
sudo apt update

# Install Ansible
sudo apt install ansible

# Verify installation
ansible --version

CentOS/RHEL/Fedora

# Install EPEL repository (if not already installed)
sudo yum install epel-release

# Install Ansible
sudo yum install ansible

# Or using dnf on newer systems
sudo dnf install ansible

# Verify installation
ansible --version

macOS

# Install using Homebrew
brew install ansible

# Verify installation
ansible --version

Python pip

# Install Ansible via pip
pip install ansible

# Verify installation
ansible --version

Setup SSH Key Authentication

# Generate SSH key pair (if not exists)
ssh-keygen -t rsa -b 4096

# Copy public key to managed nodes
ssh-copy-id user@managed-node

# Test connection
ssh user@managed-node

Essential Ansible Commands

Inventory Management

# Test connectivity to all hosts
ansible all -m ping

# Test connectivity to specific host
ansible target-host -m ping

# Test connectivity to group
ansible webservers -m ping

# List all hosts in inventory
ansible all --list-hosts

Ad-hoc Commands

# Execute shell command on all hosts
ansible all -m shell -a "uptime"

# Install package on Ubuntu/Debian
ansible webservers -m apt -a "name=nginx state=present"

# Install package on CentOS/RHEL
ansible webservers -m yum -a "name=nginx state=present"

# Copy file to remote hosts
ansible webservers -m copy -a "src=/local/file dest=/remote/path"

# Create directory
ansible webservers -m file -a "path=/opt/myapp state=directory"

# Restart service
ansible webservers -m service -a "name=nginx state=restarted"

Playbook Execution

# Run playbook
ansible-playbook playbook.yml

# Run playbook with verbose output
ansible-playbook playbook.yml -v

# Run playbook on specific hosts
ansible-playbook playbook.yml --limit webservers

# Run playbook with extra variables
ansible-playbook playbook.yml -e "env=production"

# Check playbook syntax
ansible-playbook playbook.yml --syntax-check

# Dry run (check mode)
ansible-playbook playbook.yml --check

Configuration & Debugging

# Show Ansible configuration
ansible-config dump

# Show available modules
ansible-doc -l

# Show module documentation
ansible-doc yum
ansible-doc apt

# Show playbook execution with maximum verbosity
ansible-playbook playbook.yml -vvv

Quick Start Example

Install and configure Nginx on web servers:

1. Create Inventory File (hosts.ini)

[webservers]
web1 ansible_host=192.168.1.10
web2 ansible_host=192.168.1.11

[webservers:vars]
ansible_user=ubuntu
ansible_ssh_private_key_file=~/.ssh/id_rsa

2. Create Playbook (nginx.yml)

---
- name: Install and configure Nginx
  hosts: webservers
  become: yes

  tasks:
    - name: Update package cache
      apt:
        update_cache: yes
      when: ansible_os_family == "Debian"

    - name: Install Nginx
      apt:
        name: nginx
        state: present
      when: ansible_os_family == "Debian"

    - name: Start and enable Nginx
      service:
        name: nginx
        state: started
        enabled: yes

    - name: Copy custom index.html
      copy:
        content: |
          <h1>Welcome to Nginx</h1>
          <p>Configured by Ansible</p>
        dest: /var/www/html/index.html

3. Run the Playbook

ansible-playbook nginx.yml

YAML Playbook Structure

Basic Playbook Anatomy

---
- name: Play description
  hosts: target_hosts
  become: yes/no                    # Run with sudo
  gather_facts: yes/no             # Gather system info

  vars:                           # Variables
    package_name: nginx
    service_name: nginx

  tasks:                          # List of tasks
    - name: Task description
      module_name:
        parameter1: value1
        parameter2: value2
      when: condition               # Conditional execution
      register: result             # Store result

  handlers:                       # Triggered by tasks
    - name: Restart service
      service:
        name: "{{ service_name }}"
        state: restarted
      listen: restart_webserver
💡 Tip: Use ansible-playbook --check to validate playbooks before running them in production.

Core Concepts

Understanding key Ansible components:

  • Inventory - List of managed hosts and their connection details
  • Playbooks - YAML files defining automation tasks
  • Tasks - Individual actions executed by modules
  • Modules - Reusable code units for specific operations
  • Roles - Reusable collections of playbooks and related files
  • Variables - Dynamic data used in playbooks
  • Templates - Jinja2 templates for configuration files
  • Facts - Information gathered about managed hosts
