Ansible Best Practices

This comprehensive guide covers best practices for writing maintainable, secure, and efficient Ansible automation. Following these practices will help you create production-ready infrastructure as code.

🎯 Key Principles:
  • DRY (Don't Repeat Yourself) - Avoid code duplication
  • Idempotency - Playbooks should be safe to run multiple times
  • Readability - Code should be self-documenting
  • Security - Never store secrets in plain text
  • Testability - Write testable and verifiable code

Project Organization

Directory Structure

Standard Ansible Project Layout

✅ Recommended Structure: 
my-ansible-project/
├── ansible.cfg              # Ansible configuration
├── inventories/             # Inventory files
│   ├── development/
│   ├── staging/
│   └── production/
├── group_vars/              # Group variables
│   ├── all.yml
│   └── webservers.yml
├── host_vars/               # Host variables
│   └── db01.example.com.yml
├── playbooks/               # Playbook files
│   ├── deploy-webapp.yml
│   └── setup-monitoring.yml
├── roles/                   # Roles directory
│   ├── requirements.yml     # Role dependencies
│   └── webapp/
│       ├── defaults/
│       ├── vars/
│       ├── tasks/
│       ├── templates/
│       ├── files/
│       ├── handlers/
│       └── meta/
├── library/                 # Custom modules
├── filter_plugins/          # Custom filters
├── tests/                   # Test files
│   ├── molecule/            # Molecule scenarios
│   └── test-requirements.txt
├── docs/                    # Documentation
├── README.md                # Project documentation
└── .gitignore
❌ Avoid:
  • Flat file structure with everything in root
  • Mixing inventory, playbooks, and roles in same directory
  • Hardcoded paths and values

Naming Conventions

Consistent Naming Standards

File and Directory Names:
✅ Use:
  • Lowercase with underscores: deploy_webapp.yml
  • Descriptive names: setup_nginx.yml
  • Role names: nginx, postgresql
❌ Avoid:
  • CamelCase: deployWebApp.yml
  • Abbreviations: dep_webapp.yml
  • Special characters: nginx-setup!.yml
Variable Names:
✅ Use: 
# Good variable names
nginx_port: 80
app_database_host: "{{ groups['databases'][0] }}"
ssl_certificate_path: /etc/ssl/certs/app.crt

# Consistent prefixes
app_name: mywebapp
app_version: "1.2.3"
app_port: 8080
❌ Avoid: 
# Bad variable names
port: 80                    # Too generic
dbhost: localhost           # Inconsistent naming
APP_NAME: mywebapp          # All caps
nginx_port_num: 80          # Redundant

Playbook Best Practices

Task Organization

Writing Clear and Maintainable Tasks

✅ Good Task Practices: 
tasks:
  - name: Install Nginx web server
    package:
      name: nginx
      state: present
    when: ansible_os_family == "Debian"

  - name: Configure Nginx virtual host
    template:
      src: nginx-vhost.conf.j2
      dest: /etc/nginx/sites-available/app
      mode: '0644'
      backup: yes
    notify: reload nginx
    tags: ['nginx', 'config']

  - name: Enable Nginx site
    file:
      src: /etc/nginx/sites-available/app
      dest: /etc/nginx/sites-enabled/app
      state: link
    notify: reload nginx
    tags: ['nginx', 'config']
❌ Avoid These Patterns: 
tasks:
  # Too vague
  - name: Do stuff
    shell: wget -qO- http://example.com/script.sh | bash

  # Too long command lines
  - name: Install and configure everything
    shell: |
      apt-get update && apt-get install -y nginx php mysql-server &&
      systemctl enable nginx && systemctl start nginx

  # Missing error handling
  - name: Copy files
    copy:
      src: "{{ item.src }}"
      dest: "{{ item.dest }}"
    loop:
      - { src: file1, dest: /tmp/file1 }  # No mode specified

Variable Management

Effective Variable Usage

Variable Precedence Order:
  1. Extra vars (highest priority)
  2. Task vars
  3. Block vars
  4. Role vars
  5. Play vars
  6. Host vars
  7. Group vars
  8. Defaults (lowest priority)
✅ Good Variable Practices: 
# Use defaults for flexibility
# roles/nginx/defaults/main.yml
---
nginx_port: 80
nginx_user: nginx
nginx_worker_processes: auto

# Override in inventory for specific environments
# inventories/production/group_vars/webservers.yml
---
nginx_port: 443
nginx_ssl_enabled: true

# Use facts for dynamic values
- name: Create user with correct shell
  user:
    name: appuser
    shell: "{{ ansible_facts['user']['shell'] }}"
💡 Variable Tips:
  • Use default() filter for optional variables
  • Group related variables with consistent prefixes
  • Document variable purpose and expected values
  • Avoid overriding important variables unexpectedly

Error Handling

Robust Error Handling and Recovery

✅ Implement Error Handling: 
tasks:
  - name: Check service status
    command: systemctl is-active nginx
    register: nginx_status
    ignore_errors: yes
    changed_when: false

  - name: Start nginx if not running
    service:
      name: nginx
      state: started
    when: nginx_status.rc != 0

  - name: Verify nginx is responding
    uri:
      url: http://localhost/health
      status_code: 200
    register: health_check
    until: health_check.status == 200
    retries: 30
    delay: 10
    when: nginx_status.rc == 0
❌ Avoid Silent Failures: 
tasks:
  # Dangerous - ignores all errors
  - name: Risky operation
    shell: rm -rf /important/data
    ignore_errors: yes  # Never do this!

  # Better approach
  - name: Safe file removal
    file:
      path: /important/data
      state: absent
    register: removal_result

  - name: Warn about data removal
    debug:
      msg: "Data directory removed successfully"
    when: removal_result.changed

Role Development

Role Structure

Well-Organized Role Architecture

✅ Role Best Practices: 
# roles/nginx/tasks/main.yml - Keep it simple
---
- name: Include OS-specific tasks
  include_tasks: "{{ ansible_os_family }}.yml"

# roles/nginx/tasks/Debian.yml
---
- name: Install Nginx (Debian)
  apt:
    name: nginx
    state: present
    update_cache: yes

# roles/nginx/tasks/RedHat.yml
---
- name: Install Nginx (RedHat)
  yum:
    name: nginx
    state: present

# roles/nginx/defaults/main.yml - Sensible defaults
---
nginx_port: 80
nginx_user: nginx
nginx_worker_processes: auto
nginx_ssl_enabled: false

# roles/nginx/vars/main.yml - Internal variables
---
nginx_conf_path: /etc/nginx/nginx.conf
nginx_sites_available: /etc/nginx/sites-available
nginx_sites_enabled: /etc/nginx/sites-enabled

Role Dependencies

Managing Role Dependencies

# roles/nginx/meta/main.yml
---
dependencies:
  - role: common
    tags: ['common']
  - role: ssl
    when: nginx_ssl_enabled | default(false)
    tags: ['ssl']
  - role: logrotate
    when: nginx_logrotate_enabled | default(true)
    tags: ['logging']

# roles/requirements.yml - External dependencies
---
roles:
  - name: geerlingguy.nginx
    version: "3.0.0"
  - name: geerlingguy.certbot
    version: "2.0.0"

collections:
  - name: community.general
  - name: community.crypto
💡 Dependency Tips:
  • Use ansible-galaxy install -r requirements.yml to install dependencies
  • Specify versions to ensure reproducible deployments
  • Use tags to control when dependencies are applied
  • Document role requirements in README.md

Security Practices

Secret Management

Secure Handling of Sensitive Data

✅ Security Best Practices: 
# Use Ansible Vault for secrets
# Encrypt sensitive variables
ansible-vault encrypt group_vars/production/vault.yml

# group_vars/production/vault.yml (encrypted)
---
db_password: verysecretpassword
api_key: mysecretapikey
ssl_private_key: |
  -----BEGIN PRIVATE KEY-----
  MIIEvgIBADANBgkqhkiG9w0BAQEFAASCBKgwggSk...
  -----END PRIVATE KEY-----

# Use vault variables in playbooks
tasks:
  - name: Configure database
    template:
      src: config.php.j2
      dest: /var/www/config.php
    vars:
      db_pass: "{{ db_password }}"
      api_key: "{{ api_key }}"

# Never expose secrets in logs
- name: Run sensitive command
  command: myapp --password "{{ db_password }}"
  no_log: true
⚠️ Security Warnings:
  • Never commit unencrypted secrets to version control
  • Use no_log: true for sensitive tasks
  • Rotate secrets regularly
  • Limit vault password access
  • Use different vaults for different environments

SSH Security

Secure SSH Configuration

# ansible.cfg - Secure SSH settings
[ssh_connection]
ssh_args = -o ControlMaster=auto -o ControlPersist=60s -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null
control_path = ~/.ssh/ansible-%%r@%%h:%%p
pipelining = True

# Use dedicated Ansible user
# inventories/production/hosts.ini
[webservers]
web1 ansible_host=192.168.1.10 ansible_user=ansible
web2 ansible_host=192.168.1.11 ansible_user=ansible

[webservers:vars]
ansible_ssh_private_key_file=~/.ssh/ansible_production_key
ansible_become=yes
ansible_become_method=sudo
ansible_become_user=root
🔐 SSH Security Tips:
  • Use SSH keys instead of passwords
  • Disable password authentication on managed hosts
  • Use dedicated Ansible user with minimal privileges
  • Rotate SSH keys regularly
  • Use SSH config file for connection settings

Efficient Execution

Optimizing Ansible Performance

# ansible.cfg - Performance settings
[defaults]
forks = 50                    # Parallel execution
host_key_checking = False     # Skip host key checking
gathering = smart            # Smart fact gathering
fact_caching = jsonfile       # Cache facts
fact_caching_connection = ~/.ansible/cache

[ssh_connection]

- name: Run database migration
  command: php artisan migrate
  async: 300
  poll: 10

# Selective fact gathering
- name: Gather minimal facts
  setup:
    gather_subset:
      - network
      - hardware
      - virtual

- name: Deploy application
  include_role:
    name: webapp
  tags: ['deploy']

  include_role:
    name: monitoring
  tags: ['monitoring']

Idempotency

Ensuring Safe Repeated Execution

✅ Idempotent Patterns: 
# Use state: present/absent
- name: Install packages
  package:
    name: "{{ item }}"
    state: present
  loop:
    - nginx
    - curl
    - vim

# Use creates/removes for commands
- name: Download file if not exists
  get_url:
    url: http://example.com/file.tar.gz
    dest: /tmp/file.tar.gz
  register: download_result

- name: Extract archive
  unarchive:
    src: /tmp/file.tar.gz
    dest: /opt/app
    creates: /opt/app/index.html  # Skip if already extracted

# Use check mode support
- name: Safe template deployment
  template:
    src: config.j2
    dest: /etc/app/config.yml
    backup: yes
  check_mode: yes
❌ Non-Idempotent Commands: 
# Dangerous - runs every time
- name: Add line to file
  shell: echo "nameserver 8.8.8.8" >> /etc/resolv.conf

# Better - idempotent
- name: Configure DNS
  lineinfile:
    path: /etc/resolv.conf
    line: "nameserver 8.8.8.8"
    state: present

  name: docker

    image: ubuntu:20.04

# molecule/default/verify.yml
---
- name: Verify
  hosts: all
  gather_facts: no
  tasks:
    - name: Check nginx service
      service:
        name: nginx
        state: started
      register: service_status

    - name: Verify nginx port
      wait_for:
        port: 80
        timeout: 10

    - name: Test nginx response
      uri:
        url: http://localhost
        status_code: 200

Documentation

Inline Documentation

Self-Documenting Code

✅ Good Documentation Practices: 
---
# Playbook: Deploy Web Application
# Description: Complete deployment of web application including database,
#              web server, and monitoring setup
# Author: DevOps Team
# Version: 1.2.0

- name: Deploy web application stack
  hosts: all
  become: yes

  vars:
    # Application configuration
    app_name: mywebapp
    app_version: "1.2.3"
    app_port: 8080

    # Database configuration
    db_name: mywebapp
    db_user: appuser

  pre_tasks:
    - name: Validate required variables
      assert:
        that:
          - app_name is defined
          - app_version is defined
        fail_msg: "Required variables not defined"

  roles:
    # Infrastructure setup
    - role: common
      tags: ['common']

    # Database layer
    - role: postgresql
      db_name: "{{ db_name }}"
      db_user: "{{ db_user }}"
      tags: ['database']

    # Application layer
    - role: webapp
      app_name: "{{ app_name }}"
      app_version: "{{ app_version }}"
      app_port: "{{ app_port }}"
      tags: ['application']

    # Monitoring layer
    - role: monitoring
      when: monitoring_enabled | default(true)
      tags: ['monitoring']

  post_tasks:
    - name: Verify deployment
      uri:
        url: "http://localhost:{{ app_port }}/health"
        status_code: 200
      tags: ['verify']
